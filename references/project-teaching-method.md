# Project-Based Teaching Design Method

## Learner Level Calibration

Use the learner level to control project complexity.

| Level | Project Focus | Task Granularity | Knowledge Depth | Assessment Emphasis |
| --- | --- | --- | --- | --- |
| 中职 | Concrete operation, procedure, safety, imitation to independent completion | Smaller tasks with visible outputs | Necessary concepts and operating principles | Standard operation, safety, product completion |
| 高职 | Work process, troubleshooting, tool selection, quality control | Workstation or job-role tasks | Principles linked to decisions and diagnostics | Skill performance, process documentation, quality |
| 本科 | System design, analysis, optimization, transfer, innovation | Integrated tasks with analysis and design | Conceptual model, method rationale, comparison | Design reasoning, evidence, optimization, reflection |

## Project Conversion Heuristics

Transform traditional content by asking:

- What real product, service, case, or workplace problem uses this content?
- What final artifact can students produce and show?
- What work process naturally orders the knowledge?
- What mistakes reveal whether students truly understand it?
- What equipment, software, data, materials, or constraints make the project authentic?

Good projects should be authentic, feasible within teaching time, assessable, and able to carry the required knowledge.

## Task Decomposition Pattern

Use a work-process sequence:

1. Understand the requirement or scenario.
2. Plan or design the solution.
3. Prepare tools, data, materials, or environment.
4. Execute core operations.
5. Test, inspect, debug, or evaluate.
6. Improve, document, and present.

For short units, merge adjacent stages. For complex units, split execution and testing into multiple tasks.

## Skill Analysis Pattern

Define skills as observable performance, not abstract qualities.

Skill statement formula:

`action verb + object + condition/standard`

Examples:

- Configure PLC I/O addresses according to the wiring diagram.
- Select cutting parameters based on material and tool constraints.
- Interpret sensor waveforms to diagnose abnormal signals.
- Compare two design schemes using explicit evaluation indicators.

Classify skills when helpful:

- Cognitive skill: analyze, judge, compare, calculate, model.
- Operation skill: install, connect, measure, configure, debug.
- Digital skill: simulate, search, process data, visualize, document.
- Professional skill: communicate, cooperate, follow safety rules, reflect.

## Key And Difficult Skill Point Display Pattern

After task-skill mapping, extract 4-8 key or difficult skill points for explicit teaching display.

Mark skill points consistently:

- `★` for key skills that directly determine project completion quality.
- `▲` for difficult skills that students commonly fail to perform, transfer, or explain.

For each skill point, show:

- Skill point name.
- Skill type: cognitive, operation, digital, professional, or integrated.
- Related task and work scenario.
- Performance standard in observable terms.
- Demonstration steps.
- Student practice activity.
- Common failure and likely cause.
- Coaching strategy.
- Evidence for mastery.

Good skill point displays help teachers know what to demonstrate and help students know what performance counts as competent.

## Knowledge Support Pattern

Link knowledge to skill decisions.

For every skill, identify:

- Concept knowledge: terms, definitions, components, classifications.
- Principle knowledge: why something works, cause-effect relations.
- Procedure knowledge: operation sequence and parameters.
- Standard knowledge: safety, quality, norms, acceptance criteria.
- Strategy knowledge: troubleshooting, optimization, transfer.

Mark key knowledge with `★` when it is central to completing the task.
Mark difficult knowledge with `▲` when students commonly misunderstand it or need abstraction.

## Key Knowledge Explanation Pattern

After task-knowledge mapping, extract 4-10 key or difficult knowledge points for explicit teaching display.

Mark knowledge points consistently:

- `★` for key knowledge that students must use to complete the project.
- `▲` for difficult knowledge that involves abstraction, misconception, transfer, or error-prone reasoning.

Explain each key or difficult knowledge point with:

- One-sentence plain explanation.
- Concept, principle, method/procedure, and application condition.
- A concrete task example.
- A comparison between wrong thinking and correct thinking.
- One teacher demonstration or analogy.
- One student micro-task.
- One formative question or quick quiz.
- One teaching breakthrough strategy.
- One evidence item showing that the student can apply the knowledge.

## Mind Map Rules

Skill mind maps should be generated as Image2 image assets and should move from project to tasks to observable skills.

Knowledge mind maps should be generated as Image2 image assets and should move from project to knowledge families to key and difficult points.

Use markers consistently:

- `★` for key knowledge.
- `▲` for difficult knowledge.
- `→` for transfer to skill.

Keep a text outline fallback for accessibility, but use the generated image as the primary webpage presentation.

## Teaching Animation Rules

Animations are for breaking through misconceptions, not decoration.

Use the Remotion plugin to produce final teaching animation assets. A useful teaching reference animation should:

- Show a dynamic relationship, sequence, comparison, or cause-effect process.
- Let students pause, replay, or step through.
- Pair visual change with short teacher narration.
- End with an interaction question or mini decision.

Prefer:

- Timeline animation for process knowledge.
- Before/after comparison for misconception correction.
- Flow animation for signal, force, data, energy, information, or material movement.
- Parameter slider for abstract formulas or relationships.

For each Remotion animation, specify the composition name, duration, scene sequence, motion logic, on-screen text, narration script, and rendered asset path for insertion into the final webpage.

## Assessment Design Rules

Assessment must align with outputs and performance evidence.

Use a task rubric with:

- Evidence: what students submit or demonstrate.
- Criteria: quality, accuracy, completeness, safety, standard compliance.
- Levels: Excellent, Qualified, Needs Improvement.
- Score: suggested points.
- Feedback: teacher comments or peer review prompts.

Include both process evidence and final product evidence. For vocational contexts, include safety and workplace norms whenever relevant.
